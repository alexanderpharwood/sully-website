The build.json file

Every project contains a buid.json file. It's what tells the Sully compiler how to build the project. Here is an example:

{

    "builds": {

        "production": {

            "output": "app/build/build.prod.js"

        },

        "development": {

            "output": "app/build/build.dev.js"

        }

    },

    "router": "app/router.js",

    "controllers": {

        "index": "app/controllers/indexController.js",

        "error": "app/controllers/errorController.js"

    },

    "views": {

        "header": "app/views/header.html",

        "404": "app/views/404.html",

        "403": "app/views/403.html",

        "index": "app/views/index.html",

    },

    "middleware": {

        "example": "app/middleware/ExampleMiddleware.js"

    },

    "serveIgnoreRoutePaths": [

        "vendor",

        "app",

    ]

}

Within "builds" you should specify a location for the output of production and development builds. These are the locations that the "sully build" command, and the autobuilder, will compile to. The "router" element should contain the location of your router. The "controllers", "views", and "middleware" elements should contain a list of their respective parts. The key for each is the name (as used in your router). The "serveIgnoreRoutePaths" element should be a list of all paths you want "sully serve" to ignore, i.e, to not redirect to index.html. These might be directories containing static content.
